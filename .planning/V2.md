## Part A — Examples overhaul (content)

### A1. New data shape (replaces `universalExample` for every one of the 74 frameworks)

```js
examples: [        // EXACTLY 5 entries, one per persona, order fixed as below
  { persona: "everyday",      scenario: "...", tradeoff: "..." },
  { persona: "student",       scenario: "...", tradeoff: "..." },
  { persona: "relationship",  scenario: "...", tradeoff: "..." },
  { persona: "high-achiever", scenario: "...", tradeoff: "..." },
  { persona: "privileged",    scenario: "...", tradeoff: "..." }
],
featured: <index 0-4>   // the "final example" chosen after countering — the one shown first
```

### A2. The five fixed personas (every framework gets all five)

1. **everyday** — an average working person (nurse, teacher, delivery driver, shop manager, call-center worker). Money and time pressure are REAL: rent, EMIs, school fees, one salary. No exotic vocabulary.
2. **student** — a student or early-career twenty-something: low leverage, exams, first job offers, first money mistakes, identity still forming.
3. **relationship** — a personal-life context: marriage, breakup, parenting, aging parents, in-laws, close friendships, money-between-family. The framework applied to PEOPLE, not projects.
4. **high-achiever** — a smart, driven professional/founder/specialist whose own intelligence creates the trap (overanalysis, optionality addiction, prestige chasing, "I can outwork this").
5. **privileged** — an extraordinarily wealthy or powerful person for whom money is NOT the constraint — attention, trust, meaning, succession, and reputation are. Shows the framework still binds when resources don't.

### A3. Quality bar — what "hard-hitting" and "cannot be countered" mean (testable)

Each scenario MUST:
- Name concrete stakes: a number, a deadline, a named tension (₹/$/years/a person). No "imagine you have a task" phrasing.
- Show the framework FLIP or sharpen the decision — a before-thought and an after-thought, not a definition restated in story clothes.
- Be about a decision the persona plausibly faces THIS YEAR, not a thought experiment.

Each tradeoff MUST:
- State what the framework-guided choice genuinely COSTS (no free lunches; "no downside" is a defect).
- Pre-empt the strongest counterargument — the steelman objection is absorbed into the tradeoff, which is the operational meaning of "cannot be countered". (Example shape: "You may kill a project that would have turned; the Stop Rule trades a small number of false quits for immunity to the sunk-cost slide.")

Authoring process (first principles, enforced by trace evidence): for each framework the Generator drafts candidate examples, ATTACKS each with the strongest counter-logic ("would a skeptic say the framework didn't matter here / the persona wouldn't act this way / the stakes are fake?"), revises or discards, then selects `featured` — the single example that best survives. The generator trace must show this draft→counter→final loop for at least 5 sample frameworks in full, and assert it was applied to all.

### A4. personalPrompt upgrade

Every framework's `personalPrompt` must be re-audited: it should be allowed (and often expected) to touch personal life — relationships, family, money, identity, health — not just work/projects. Rewrite any prompt that only addresses tasks/projects if the framework naturally applies to persons (most do).

### A5. Copyright rules unchanged

All new text is original authoring from the concept itself. RESEARCH.md ground rules still bind: never quote the book; trigger/essence fields remain byte-stable (B2).

## Part B — UI transformation (design)

The v1 UI passed its gates but reads as a competent generic dark app. V2 must read as a DIFFERENT, considered product. Direction (observable criteria for contracts):

- **New visual identity**: replace the v1 palette and type system. Direction: "pocket field-manual / editorial print" — a paper-warm light theme and a deep-ink dark theme, a display face for framework names distinct from the body face, generous margins, visible grid. The v1 CSS custom-property values for core surface/accent tokens and the font stack must actually CHANGE (testable proxy for "transformed").
- **Example presentation**: the five persona examples are the heart of the card — featured example shown prominently with its tradeoff visually paired (scenario → "the cost" label), other four behind one tap (persona-labelled tabs/accordion with distinct persona glyphs/colors). Tradeoff is NEVER hidden when its scenario is visible.
- **Persona system**: the five personas get a consistent, named visual identity (label + accent) reused across every card.
- **Hierarchy**: essence stays above the fold at 375px; diagram larger than v1; pitfalls visually distinct from examples; card still ENDS on the personal prompt question.
- **Keep**: bottom tab navigation, offline behavior, all v1 behaviors (B1–B23), AA contrast both themes, reduced-motion respect, 375px no-horizontal-scroll, ≥44px targets.
- **Anti-goals**: no framework/library adoption, no raster imagery, no lorem-ish placeholder styling, no loss of any v1 test guarantee.
