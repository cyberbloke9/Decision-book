## Ground rules (verified)

1. **Copyright**: Concepts/frameworks/methods are NOT copyrightable (17 U.S.C. §102(b); Baker v. Selden 1879; TRIPS art. 9(2)). The book's *text and illustrations* ARE protected (published 2011, Profile Books). No safe-harbor word count exists; paraphrasing/restyling does not create ownership. → **Every sentence and every SVG in the app must be written/drawn from scratch.** Standard diagram forms (2x2 matrix, quadrants) fall under the merger doctrine — free to use. Model NAMES are fine to use.
2. **Book DNA to preserve** (verified from authors' own intro): every model must (a) simplify, (b) sum up, (c) be visual, (d) be a *method that asks questions rather than gives answers*. The book is a workbook — models are meant to be filled in and modified. → App cards should end in a question/action, not a lecture.
3. **Book structure** (verified 3–0 from publisher TOC): fifty models in four sections: How to Improve Yourself / How to Understand Yourself Better / How to Understand Others Better / How to Improve Others. Every TOC entry pairs the model with a use-case subtitle → maps directly to the app's trigger field. ~3–4 pages per model in a 186-page book → one-model-one-screen is faithful.

## Extension Set B — Cognitive biases with catch-yourself cues (Decision Lab format: hook question + everyday example + cue)

| Bias | Catch-yourself cue | Essence | Visual |
|------|--------------------|---------|--------|
| Sunk Cost Fallacy | "…but I've already put so much in" ← that sentence is the alarm | Past investment is gone either way; judge only future costs vs future benefits | sinking anchor |
| Confirmation Bias | You feel *satisfied* by evidence | You seek what agrees with you. Cue: actively hunt one disconfirming fact before deciding | filter funnel |

## Habit-embedding mechanics (for the Daily Card + triggers design)

- Reading ≠ learning: success criterion is recall in a real scenario → cards must test application ("which model fits this situation?"), not just definitions.
- Per-concept schema that works for spaced learning: name / description / visual / implication / drawbacks — matches our card fields exactly.
- Spacing: expanding intervals (~1d, 4d, 12d, 1mo, 3mo) timed near the forgetting point; v1 daily-card rotation is the simple version, SRS is v2.
- Habit automaticity: median 66 days (Lally 2010, range 18–254) → streak UI should celebrate persistence, not punish misses.
- Low-effort passive review (re-showing a card) is worse than testing but still valuable and sustainable when energy is low → daily card should offer both glance mode and a one-tap "apply it" reflection.
- Real-time recall is THE bottleneck for frameworks like WRAP → the situation-triggered navigation ("I'm stuck", "can't choose") is the core differentiator, mirroring implementation-intentions logic (cue → response).
